Term Website Project

The term website project is designed to be a comprehensive assessment of all the course objectives. You will be given specific specifications as if you were contracted to plan, design, and build a prototype website for a real-world organization. In addition, developmental specifications will be given to ensure proper assessment of the application of course content, skills, and technologies.  This independent, term project will be of your own design and work, however, you are encouraged to work with your peers on design and development issues. The content is fictitious but the site should be a complete working prototype containing relevant and reasonable information. The images and brief verbiage may be referenced from other sources which will need to be cited in a resource reference page.

Company Description

The Salmon River Adventures company is located in Riggins, Idaho and offers world-class, whitewater rafting experiences on the Salmon River in the middle of an Idaho wilderness. The owner-operated company offers day and overnight trips for groups and individuals using state-of-the-art equipment and complies with all U.S. Forest Service regulations. They are looking for a contemporary, inviting website build that reflects their core business values of high-end adventures, professional and experienced service, safety, and fun.

Specifications

The following design and content specific items are required in order to meet the functional specifications of Salmon River Adventures.

  • The site must be responsive to small, medium, and larger, wide screen views.
  • The design is consistent throughout the website providing ease of navigation and supporting excellent user experiences.
  • The home page is a contemporary, exciting landing page providing images of the river, whitewater rafting, kayaking, and/or related activities and including a call to action and readily available navigation and contact information. The home page should also display the current local weather conditions in Riggins, Idaho. Use a service like the wunderground api.
  • Include three distinct adventure trip / activity pages that describe the length, skill levels required, costs, and services provided by the company.
  • Include one "River Guide" page with a minimum of three profiles about the outstanding, professional employees of the company including their name, picture, certification level, years of experience, email address, and a brief biography sketch.
  • The contact page will contain essential contact information including an embedded google map showing the company address at 1000 S Main St., Riggins, ID and a basic contact us form. (The form does not need to process anything.)
  • A reservation page that contains a data table summarizing the three adventures with length, skill level, and cost information and a registration form that requests the customer's full name, email, phone, adventure selection, start date selection, and a question/comment text area. (This form does not need to process the input.)

Developmental Specifications

The following development specifications are required in addition to the customer specifications in order to receive full credit on the term website project.

  • Structure the site with valid, standards-based, semantic markup that is responsive to small, medium and large screen views.
  • Ensure that all browsers render elements consistently by referencing a normalization CSS on all pages.
  • Support HTML5 features on older browsers using a modernizr script on all pages.
  • Modulate the markup for at least three repeatable sections of the website using PHP technology.  
  • Design a logo to be used at appropriate levels and locations throughout the website.
  • Follow PARC principles and typography best-practices throughout the website.
  • Implement an appropriate font, of your choice, using a fonts.google.com reference.
  • Implement an accordion menu or a horizontal slide experience to display the guide profiles one at a time on the river guide page using jQuery.
  • Drive the adventure data on the registration page using an external data file source that is written in JSON format.
  • Ensure that each page weight does not exceed the 400k benchmark.
  • Ensure that there are no spelling or grammatical errors.
  • Publish the complete, working site on your own domain in the web root or on a sub-domain.
  • Include a resource page that references outside content if necessary in the footer.
  • Create a site overview video that provides a brief (~5-7 min), narrated overview of the site with references to how the site meets the needs of the identified target audience, follows PARC principles, meets the developmental specifications and is responsive. This video should be placed in separate page using HTML5 video element and linked in the footer of your website.

NOTE:  This is a web course intended to help you learn how to construct websites using the core web technologies of contemporary HTML, CSS and JavaScript. Outside templates are NOT allowed. Pages built from site builder software or drag-n-drop tools or that are based on existing sites are not allowed and will lead to a failing grade on the term website project.

Assessment

All course objectives and documented specifications are assessed in the term website project. Use the following matrix to guide your development and testing before final submission. The point allocation on the right of each statement indicates the maximum points possible.
